# ADR-001 — sealed record para Value Objects

** STATUS ** :  Proposed
**DATE** : 2024-06-17
**Deciders:** Carlos Emmanuel 

## Contexto

EXISTE UNA VALIDACION DE PRECIO EN LA CLASE DE SERVICIO DE PRODUCTOS, ESTA VALIDACION SE REALIZA EN EL METODO DE CREACION Y ACTUALIZACION DE PRODUCTOS,
ESTA VALIDACION ES REUTILIZABLE Y SE PUEDE ABSTRAER PARA SER UTILIZADA EN OTRAS PARTES DEL SISTEMA.


## DECISION
SE DECIDE DIVIDIR LA VALIDACION DE PRECIO EN UNA CLASE SEPARADA, ESTA CLASE SE ENCARGARA DE REALIZAR LA VALIDACION DE PRECIO Y SE PODRA UTILIZAR EN CUALQUIER PARTE DEL SISTEMA QUE REQUIERA VALIDAR PRECIOS.

## CONSECUENCIAS 
POSITIVAS: 
- SE MEJORA LA REUTILIZACION DE CODIGO, YA QUE LA VALIDACION DE PRECIO SE ENCUENTRA EN UNA CLASE SEPARADA Y PUEDE SER UTILIZADA EN CUALQUIER PARTE DEL SISTEMA.
- SE MEJORA LA MANTENIBILIDAD DEL CODIGO, YA QUE LA VALIDACION DE PRECIO SE ENCUENTRA EN UNA CLASE SEPARADA Y SE PUEDE MODIFICAR SIN AFECTAR OTRAS PARTES DEL SISTEMA.
- SE MEJORA LA LEGIBILIDAD DEL CODIGO, YA QUE LA VALIDACION DE PRECIO SE ENCUENTRA EN UNA CLASE SEPARADA Y SE PUEDE ENTENDER MEJOR SU FUNCIONALIDAD.
- SE MEJORA LA TESTABILIDAD DEL CODIGO, YA QUE LA VALIDACION DE PRECIO SE ENCUENTRA EN UNA CLASE SEPARADA Y SE PUEDE PROBAR DE MANERA INDEPENDIENTE.

NEGATIVAS: 
- SE AUMENTA LA COMPLEJIDAD DEL SISTEMA, YA QUE SE CREA UNA CLASE SEPARADA PARA LA VALIDACION DE PRECIO, LO QUE PUEDE HACER QUE EL SISTEMA SEA MAS DIFÍCIL DE ENTENDER PARA LOS DESARROLLADORES QUE NO ESTAN ACOSTUMBRADOS A ESTE TIPO DE ARQUITECTURA.
- SE AUMENTA EL TIEMPO DE DESARROLLO, YA QUE SE CREA UNA CLASE SEPARADA PARA LA VALIDACION DE PRECIO, LO QUE PUEDE HACER QUE EL TIEMPO DE DESARROLLO SEA MAS LARGO.
